# 📊 Origins Logic - Подробное объяснение

## 🎯 Что такое "Origin"?

**Origin** — это точка во времени, с которой мы начинаем делать прогноз.

### Пример:
```
Данные: 2020-01-01 до 2023-12-31 (1461 дней)

Origin 1: 2020-04-01
  - История: 2020-01-01 до 2020-04-01 (91 день)
  - Прогноз на 30 дней: 2020-05-01
  - Прогноз на 365 дней: 2021-03-31

Origin 2: 2020-04-02
  - История: 2020-01-01 до 2020-04-02 (92 дня)
  - Прогноз на 30 дней: 2020-05-02
  - Прогноз на 365 дней: 2021-04-01

И так далее...
```

## 📐 Логика расчёта Origins

### **Код в Prophet/SARIMA:**
```python
dates = vol.index  # Все даты в данных
start_i = 90       # Минимальная история (90 дней)
end_i = len(vol) - max_h  # Последняя возможная дата (чтобы поместился прогноз на max_h дней)

# Шаг между origins
step = max(1, (end_i - start_i) // 800)

# Примеры:
origins = range(start_i, end_i, step)
```

### **Пример расчёта:**
```python
# Данные: 1461 дней (2020-01-01 до 2023-12-31)
# max_h = 365 (самый длинный горизонт)

start_i = 90  # Начинаем с 2020-04-01 (нужно минимум 90 дней истории)
end_i = 1461 - 365 = 1096  # Заканчиваем на 2022-12-31 (чтобы поместился прогноз на 365 дней)

# Вариант 1: step = 1 (каждый день)
origins = range(90, 1096, 1)  # 1006 origins

# Вариант 2: step = 800 (каждый ~2-й день)
step = (1096 - 90) // 800 = 1
origins = range(90, 1096, 1)  # 1006 origins (step минимум 1)

# Вариант 3: Если данных меньше
# Например, 500 дней данных, max_h = 365
start_i = 90
end_i = 500 - 365 = 135
step = (135 - 90) // 800 = 0 -> max(1, 0) = 1
origins = range(90, 135, 1)  # 45 origins
```

## 🔢 От чего зависит максимальное количество Origins?

### **Формула:**
```python
max_origins = (len(data) - start_i - max_horizon) / step
```

### **Факторы:**

#### **1. Длина данных (`len(data)`)**
- Больше данных = больше возможных origins
- **Пример:**
  - 1000 дней данных → ~600 origins (при max_h=365, start_i=90)
  - 2000 дней данных → ~1600 origins (при max_h=365, start_i=90)

#### **2. Минимальная история (`start_i`)**
- Минимальное количество дней истории для обучения модели
- **Текущее значение:** 90 дней
- **Рекомендации:**
  - Prophet: минимум **365 дней** (для годовой сезонности)
  - SARIMA: минимум **90-180 дней** (для сезонных паттернов)

#### **3. Максимальный горизонт (`max_h`)**
- Самый длинный прогноз
- **Пример:** max_h = 365 означает, что последний origin должен быть минимум за 365 дней до конца данных

#### **4. Шаг (`step`)**
- Как часто брать origins
- **Текущая логика:**
  ```python
  step = max(1, (end_i - start_i) // 800)
  ```
- **Можно изменить:**
  ```python
  # Для большего количества origins:
  step = max(1, (end_i - start_i) // 1500)  # ~1500 origins
  
  # Для максимального количества (каждый день):
  step = 1  # Все возможные origins
  ```

## ⚙️ Как увеличить количество Origins?

### **Вариант 1: Уменьшить шаг (самый простой)**
```python
# Текущее (в Prophet и SARIMA):
step = max(1, (end_i - start_i) // 800)

# 🔧 УВЕЛИЧИТЬ до 1500 origins:
step = max(1, (end_i - start_i) // 1500)

# 🔧 Максимум origins (каждый день):
step = 1
```

**Плюсы:**
- ✅ Больше origins = более статистически значимые результаты
- ✅ Лучше покрытие временного ряда

**Минусы:**
- ❌ Увеличение времени выполнения пропорционально количеству origins
- ❌ Больше RAM для хранения результатов

### **Вариант 2: Уменьшить минимальную историю**
```python
# Текущее:
start_i = 90  # 90 дней

# 🔧 Уменьшить:
start_i = 60  # 60 дней (рискованно для Prophet!)

# Расчёт:
# Было: (1096 - 90) = 1006 возможных origins
# Стало: (1096 - 60) = 1036 возможных origins (+30)
```

**⚠️ Осторожно:**
- Prophet требует **минимум 365 дней** для качественного прогноза годовой сезонности
- SARIMA может работать с 60-90 днями, но с меньшей точностью

### **Вариант 3: Использовать больше данных**
```python
# Если у вас есть исторические данные за больший период
# Например, вместо 4 лет использовать 10 лет:

# Было: 1461 дней (4 года)
# Стало: 3653 дня (10 лет)

max_origins = (3653 - 90 - 365) / 1 = 3198 origins!
```

**Время:** ~30-60 минут (Prophet), ~6-12 часов (SARIMA)

**Время:** ~1-2 часа (Prophet), ~12-24 часа (SARIMA)

## 💡 Рекомендации для вашего случая

### **Для гибридной стратегии (SARIMA + Prophet):**

```python
# Prophet backtest:
start_i = 365           # 🔧 Увеличено! Prophet нужна годовая сезонность
step = max(1, (end_i - start_i) // 1000)  # ~1000 origins
horizons = [1, 2, 3, 7, 30, 90, 180, 365]

# SARIMA backtest:
start_i = 90            # SARIMA может с меньшей историей
step = max(1, (end_i - start_i) // 1200)  # ~1200 origins
horizons = [1, 2, 3, 7, 30, 90, 180, 365]
```

**Обоснование:**
1. **Prophet:** Больше истории (365 дней) для годовой сезонности
2. **SARIMA:** Больше origins (1200) для лучшей статистической значимости
3. **Одинаковые горизонты:** Для честного сравнения

### **Оценка времени выполнения:**

```python
# Prophet (1000 origins, 8 horizons):
# ~0.05 сек/origin * 1000 origins * 8 horizons = ~400 секунд = ~7 минут ✅

# SARIMA (1200 origins, 8 horizons):
# С кэшированием (95% hit rate):
# ~2 сек/origin * 1200 origins * 8 horizons * 0.05 = ~960 секунд = ~16 минут ✅
# Без кэширования:
# ~2 сек/origin * 1200 origins * 8 horizons = ~19200 секунд = ~5.3 часа ⚠️
```

## 🎯 Итоговая рекомендация для вашей задачи

**Для тестирования гибридной стратегии (SARIMA краткосрочно + Prophet долгосрочно):**

```python
# Prophet configuration:
horizons = [1, 2, 3, 7, 30, 90, 180, 365]
start_i = 365        # Годовая сезонность
step = max(1, (end_i - start_i) // 1000)
chunk_size = 100
n_jobs = -1

# SARIMA configuration:
horizons = [1, 2, 3, 7, 30, 90, 180, 365]
start_i = 90         # SARIMA эффективен с меньшей историей
step = max(1, (end_i - start_i) // 1200)
chunk_size = 400     # Больше для 80GB RAM
n_jobs = 5

# Ожидаемое время:
# - Prophet: ~10-15 минут
# - SARIMA: ~20-30 минут (с кэшированием)
```

**После получения результатов вы сможете принять решение:**
- Использовать SARIMA для 1-7 дней (если R² > 0.9)
- Использовать Prophet для 90-365 дней (если RMSE лучше SARIMA)
- Использовать ансамбль для 30-90 дней (взвешенное среднее)
