# 🔮 PROPHET BACKTEST - Критический анализ

## 📊 Проблема: Количество origins

### **Обнаруженная ошибка:**
```python
# ❌ НЕПРАВИЛЬНО (в первых версиях):
step = max(1, (end_i - start_i) // 200)  # Всего 212 origins

# ✅ ПРАВИЛЬНО (как в SARIMA):
step = max(1, (end_i - start_i) // 800)  # ~800 origins
```

**Результат:** Prophet тестировался на **212 origins**, а SARIMA на **1474** — несправедливое сравнение!

## 🔍 Критические выводы

### **1. Prophet НЕ подходит для коротких горизонтов (1-30 дней)**
- Prophet оптимизирован для **долгосрочных прогнозов** (месяцы/годы)
- Для 1-30 дней SARIMA показывает **в 10-20 раз лучшие результаты**

### **2. Prophet показывает потенциал на 365 дней**
- RMSE 94.69 vs SARIMA 224.20 — **в 2.4 раза лучше!**
- Но R² всё ещё отрицательный (-3.94), что указывает на **систематическую ошибку**

### **3. Несправедливое сравнение**
- **Prophet:** 212 origins (из-за ограничения `// 200`)
- **SARIMA:** 1474 origins
- **Нужно:** Перезапустить с 800 origins для честного сравнения

### **4. Проблема переобучения**
- `changepoint_prior_scale` критически важен:
  - 0.05: Слишком жесткий, не адаптируется
  - 0.5: Катастрофическое переобучение (R² = -514!)
  - 0.01: Оптимально для этих данных

### **5. Multiplicative seasonality важна**
- Для водохранилищ сезонность **масштабируется с объемом**
- `seasonality_mode='multiplicative'` даёт лучшие результаты
