## 3. Основной используемый метод: Theil-Sen

### Что использует модель в web-приложении:

```python
def build_robust_season_trend_series(
    hist_df: DataFrame,
    future_days: int = 180,
    season_basis: str = 'DOY',
    min_history: int = 365
) -> SeasonTrendResult:
    """
    Детерминированный прогноз:
    
    1. Вычислить сезонный медиан (для каждого DOY)
    2. Вычесть сезон → остаток
    3. Применить Theil-Sen регрессию к остатку
    4. Продлить: Season(DOY) + Trend_slope × t
    """
```

**Используемое уравнение:**
$$\text{Value}(t) = \text{Season}(\text{DOY}) + \text{Slope} \times t + \text{Residual}(t)$$

**Где:**
- **Season(DOY)** – медиан значений для каждого дня года
- **Slope** – коэффициент тренда (Theil-Sen)
- **Residual** – остаток

### Преимущества Theil-Sen:
- ✅ Робастность к выбросам (медиан вместо среднего)
- ✅ Низкие вычислительные требования
- ✅ Проста для интерпретации
- ✅ Быстрая (O(n²) вместо O(n³) для SARIMA)

## 4. Ошибки SARIMA (если включить)

Если попытаться использовать SARIMA в web-приложении, столкнётесь с проблемами:

### Исправленные ошибки:

1. **NaN в данных** ✅ Исправлено
   - Решение: Forward fill → Backward fill

2. **Out of Memory (OOM)** ✅ Исправлено
   - Решение: Ограничение истории до 730 дней (`max_history=730`)
   - **Экономия памяти:** ~73% для больших серий

3. **Адаптивная сезонность** ✅ Реализовано
   ```python
   if days >= 3*365:
       m = 365  # годовая
   elif days >= 180:
       m = 30   # месячная
   elif days >= 60:
       m = 7    # недельная
   else:
       m = 1    # без сезонности
   ```

4. **Fallback на non-seasonal** ✅ Реализовано
   - Если сезонная модель падает → простая ARIMA

- `BACKTEST_ERROR_FIXES.md` – подробное описание всех исправлений

## 5. Рекомендации

### Для текущей версии в продакшене:

✅ **Продолжайте использовать Theil-Sen** – она:
- Стабильна и предсказуема
- Не требует дополнительных зависимостей
- Быстра и экономична по памяти
- Удовлетворяет текущим потребностям

### Если нужна SARIMA:

1. **Добавить в requirements.txt:**
   ```
   pmdarima>=2.0.0
   statsmodels>=0.13.0
   ```

2. **Включить в UI:**
   ```python
   # В wbm/ui/controls.py добавить выбор метода
   forecast_method = st.radio("Метод прогноза", 
                              ["Theil-Sen (быстро)", "SARIMA (экспериментально)"])
   
   if forecast_method == "SARIMA (экспериментально)":
       result = build_sarima_model(series, future_days=future_days)
   else:
       result = build_robust_season_trend_series(series, future_days=future_days)
   ```

3. **Добавить предупреждение:**
   ```python
   st.warning("⚠️ SARIMA – экспериментальный метод. Может быть медленным на больших данных.")
   ```

### Если нужен Prophet:

1. **Установить:**
   ```
   pip install pystan==2.19.1.1
   pip install prophet
   ```

2. **Создать функцию:**
   ```python
   from prophet import Prophet
   
   def build_prophet_forecast(df, future_days=180):
       """df должен иметь столбцы: ds (дата), y (значение)"""
       model = Prophet()
       model.fit(df)
       future = model.make_future_dataframe(periods=future_days)
       forecast = model.predict(future)
       return forecast
   ```

3. **Интегрировать в UI**
