## 3. Логика модели водного баланса

Дневной шаг, детерминированное обновление объёма:
- Вход: начальный объём V0, прогноз/климат P(t) и ET(t), приток/сброс Q_in/Q_out, связь объёма и площади A(V)
- Обновление: V(t+1) = V(t) + P_eff·A − ET_eff·A + Q_in − Q_out
- Геометрия: `wbm.curve` предоставляет калиброванную кривую «объём‑площадь»
- Климат: P/ET формируются одним из методов (см. раздел 4)
- Фильтры качества: скрытие малых площадей (< min_area_km2) для визуализации

Осадки и испарение могут быть взяты:
- Как климатические среднемесячные значения (быстрый режим)
- Как сезонная климатология
- Как сезон + тренд (Theil‑Sen) или через статистические модели (SARIMA/Prophet/SARIMAX)

## 4. Методы прогнозирования P/ET

Доступны 4 метода (выбор в боковой панели UI):

1) Theil‑Sen (быстро, робастно)  
- Метод медианного наклона по тренду + сезонная компонента (DOY/MONTH)
- Преимущества: минимальные зависимости, устойчив к выбросам
- Использование: `build_robust_season_trend_series()` → `deterministic`

2) SARIMA (авторегрессионная сезонная модель)  
- Автоматический подбор через `pmdarima` с защитой от OOM
- Хорошо ловит автокорреляцию и сезонность
- Использование: `build_sarima_forecast_enhanced()`

3) Prophet (продвинутый)  
- Гибкая работа с трендами/сезонностями, интервалы
- Использование: `build_prophet_forecast()`

4) SARIMAX (SARIMA + внешние регрессоры) — НОВОЕ  
- Учитывает экзогенные переменные (температура, влажность, радиация и т.п.)
- Использование: `build_sarimax_forecast(series, exog_data=..., ...)`
- Для будущих дат экзогенные признаки по умолчанию продлеваются методом «персистентности» (последние известные значения). При наличии прогнозов по экзогенным — можно подать их в `exog_data`.

Типичные диапазоны MAPE (%), ориентиры:
```
			  1‑день  1‑неделя  1‑месяц  6‑месяцев
Theil‑Sen:    10–15   12–18     15–25    20–35
SARIMA:        8–12   10–15     13–20    18–30
Prophet:       9–13   11–16     14–22    19–32
SARIMAX:       7–11    9–14     12–18    17–28   (при качественных регрессорах)
```

## 5. Метрики точности прогноза (1д, 1н, 1м, 6м)

Реализовано в `wbm/metrics.py`:
- MAPE — средняя абсолютная процентная ошибка
- RMSE — среднеквадратичная ошибка
- MAE — средняя абсолютная ошибка
- R² — доля объяснённой вариации

Базовые функции:
- `calculate_mape(actual, predicted)`
- `calculate_rmse(actual, predicted)`
- `calculate_mae(actual, predicted)`
- `calculate_r_squared(actual, predicted)`
- `calculate_metrics_by_horizon(actual: Series, predicted: Series, horizons=[1,7,30,180])`
- `format_metrics_for_display(metrics)` для UI
- `best_method_by_horizon(metrics_by_method)` для сравнения алгоритмов

Интеграция в UI:
- `wbm/ui/simulation.py → display_forecast_metrics(actual, forecast, variable_name)` — вывод карточек метрик по горизонтам в раскрываемом блоке

## 7. Ограничения и допущения

- Разрешение по времени — дневной шаг; подчасовая динамика не моделируется
- SARIMAX: по умолчанию для будущих экзогенных признаков — персистентность (если нет отдельного прогноза)
- Качество прогноза зависит от консистентности истории и охвата сезонных циклов (желательно 2+ года)
- Батиметрия/кривая объём‑площадь — ключевой источник неопределённости; при крупных реконструкциях DEM требуется верификация
- MAPE нестабилен при нулях в фактических значениях (используется `epsilon` для избежания деления на ноль)
- Prophet/SARIMA/SARIMAX — более ресурсоёмкие; предусмотрен автоматический откат к Theil‑Sen при сбоях

## 8. Безопасность и эксплуатационные риски

- Прозрачность вычислений: все шаги детерминированы и повторяемы; версии библиотек фиксируются в `requirements.txt`
- Защита от отказов: предупреждения, перехват исключений, fallback на Theil‑Sen
- Контроль памяти/времени: ограничение истории для SARIMA, отключение навязчивых предупреждений, аккуратная работа с большими матрицами
- Данные: при работе с внешними API/GEE рекомендуется хранить локальные копии исходных файлов, сверять контрольные суммы
- Эксплуатационные риски: неверная калибровка A(V), ошибки в исходной климатологии, пропуски наблюдений — влияют на объёмы; для критически важных решений рекомендуется страховой запас и сценарный анализ

## 9. Экономический эффект (потенциальная выгода)

Примеры бизнес‑кейсов и ориентиры экономии/дохода:
- Управление водопользованием (орошение, водоснабжение): снижение недоотбора/перерасхода 3–10% благодаря более точному горизонту на 1–4 недели
- Энергетика (ГЭС): оптимизация графика сбросов под прогноз притока → до +1–3% выработки на пике/минимизация потерь
- Наводнения/переливы: раннее предупреждение (1н–1м) снижает ущерб инфраструктуре; даже 10–20% снижение вероятности критических сценариев может экономически оправдать внедрение
- Экологические режимы: поддержание минимальных экологических расходов снижает штрафные и репутационные риски
- Портфельный эффект: сценарный анализ для инвестпроектов (капвложения в дноуглубление/дамбы) с оценкой NPV при изменчивом климате

SARIMAX при наличии качественных экзогенных признаков типично снижает MAPE на 2–4 п.п. относительно SARIMA/Prophet, что прямо конвертируется в уменьшение операционных рисков и рост эффективности.
